# Control

## 当前阶段范围

当前只实现 Phase 2 的手动遥控和安全链路：

- 底盘摇杆
- 云台摇杆
- 模式切换
- 急停
- 解除急停
- 断连自动归零
- 失焦自动归零
- 心跳超时自动归零

## 控制模式

### 1. 底盘模式

- 左摇杆控制底盘
- 右摇杆输入会被记录，但不驱动底盘

### 2. 云台模式

- 右摇杆控制云台
- 底盘保持静止

### 3. 联动模式

- 左摇杆控制底盘
- 右摇杆控制云台
- 云台左右偏转会叠加低速底盘角速度，便于对准目标

## 控制链路

前端通过 WebSocket 连接：

```text
/ws/control
```

后端提供控制状态接口：

```text
/api/control/state
```

## 安全策略

1. 松开摇杆后自动归零。
2. 浏览器失焦后自动归零。
3. 页面关闭后自动归零。
4. WebSocket 断开后自动归零。
5. 心跳超时后自动归零。
6. 急停触发后忽略新的摇杆输入，直到解除急停。

## 当前限制

- 当前版本默认先验证控制状态机，不默认向真实机器人发布命令。
- 真机控制桥需要在 Ubuntu 20.04 + ROS2 Foxy 环境中继续接入。
- 当前自动发现仅识别 `cmd_vel` 和部分云台 Topic 候选，不保证覆盖所有机器人封装实现。
